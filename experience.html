My Professional Experience

Cybersecurity Intern

CyberSapiens United LLP | Feb 2025 - Present

During my internship at CyberSapiens United LLP, I've gained hands-on experience in web application penetration testing and vulnerability assessment. My primary responsibilities include identifying security flaws, documenting findings, and recommending effective remediation strategies.

Key Contributions & Learning:

  • Discovered Critical JWT Vulnerability: Identified and exploited a JWT (JSON Web Token) algorithm confusion vulnerability (RS256 to HS256). This allowed unauthorized manipulation of token parameters, leading to sensitive data modification (e.g., username change). This critical finding highlighted weaknesses in token validation and signature verification processes.
  • Identified Session Hijacking via Cookie Swapping: Uncovered a session hijacking vulnerability where an authenticated user's session could be taken over by swapping cookies. By capturing a test user's session and replacing it with an authenticated user's session from another browser, I could hijack the authenticated user's session, leading to potential account takeover and unauthorized actions.
  • Found Cross-User Cart Manipulation: Discovered a medium-to-high severity vulnerability allowing cross-user cart manipulation through JWT reuse. By replacing User B's token payload with User A's and manipulating the `aud` parameter in the cookie, an item was added to User B's cart after a refresh, despite the operation originating from User A's token.
  • Learned Advanced Vulnerability Identification & Exploitation:
    • OTP Bypass and Captcha Bypass: Learned how to bypass One-Time Password (OTP) and Captcha mechanisms to evade authentication and anti-automation controls.
    • Host Header Injections: Understood how to manipulate the `Host` header to affect server-side logic, leading to attacks like password reset poisoning or web cache deception.
    • CSRF (Cross-Site Request Forgery): Understood how malicious websites can force users' browsers to perform unwanted actions on websites where they are authenticated.
    • Business Logic Vulnerabilities: Learned to identify and exploit business logic flaws, which exist in the unique functionality of an application and are not detected by standard security tools.
    • Session Hijacking & Session Fixation: Learned to recognize and exploit session management flaws that allow attackers to control user sessions.
    • XSS (Cross-Site Scripting) & HTML Injections: Learned to exploit input validation flaws that allow malicious scripts to execute client-side or arbitrary HTML to be injected.
    • JWT Token Hacking: Learned to identify and exploit vulnerabilities in JWT tokens (such as algorithm confusion, weak keys).
    • SQL Injections: Learned to manipulate database queries to extract sensitive data or perform unauthorized actions.
    • XXE (XML External Entity), CORS (Cross-Origin Resource Sharing) & HSTS (HTTP Strict Transport Security): Explored misconfigurations and vulnerabilities in XML parsers, cross-origin policies, and secure transport mechanisms.
    • File Upload/Traversal, LFI (Local File Inclusion) & RFI (Remote File Inclusion): Learned to identify and exploit file-related vulnerabilities that can lead to arbitrary file upload, sensitive file access, or remote code execution.
    • SSRF (Server-Side Request Forgery), Subdomain Takeover and Broken Link Hijacking: Learned to manipulate server-side requests, take over expired subdomains, and exploit broken links.
    • RCE (Remote Code Execution) and Command Injection, Buffer Overflow and DoS Attacks: Learned to exploit critical vulnerabilities that can allow arbitrary code execution on the server, memory corruption, or making a service unavailable.
    • API Specific Vulnerabilities:
      • BOLA (Broken Object Level Authorization): Identified authorization flaws where a user can access unauthorized objects.
      • BOPLA (Broken Function Level Authorization): Identified authorization flaws where a user can access unauthorized functions.
      • Unrestricted Resource Consumption: Exploited lack of resource limits in APIs leading to DoS attacks or high resource usage.
      • Unrestricted Access to Sensitive Business Flows: Business logic flaws that do not adequately protect sensitive API flows.
      • Broken Authentication: Identified flaws in API authentication mechanisms.
      • SSRF (Server-Side Request Forgery) in API: Manipulated server-side requests through API endpoints.
      • Security Misconfiguration: Exploited misconfigurations such as default credentials, unpatched systems, or improper configurations.
      • Improper Inventory Management: Identified vulnerabilities arising from discrepancies in API documentation and inventory.
      • Unsafe Consumption of APIs: Identified vulnerabilities caused by client applications insecurely consuming APIs.

These findings significantly contributed to the security posture of the assessed applications. My mentors and team at CyberSapiens United LLP have been incredibly supportive, providing valuable guidance throughout this learning process.

In recognition of these contributions, I was honored as "Performer of the Month" for May 2025, receiving a certificate for my efforts.
